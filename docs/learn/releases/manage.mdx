Releases are located in the [Releases page](https://app.miruml.com/releases) of the dashboard.

## Create a release

To create a release, click the **New Release** button in the top right and supply a version. 

<Frame>
  ![](https://assets.miruml.com/docs/images/releases/create-dialog-step1.png)
</Frame>

<Info>
  The version must be a [semantic version](https://semver.org/). See the [release properties](/docs/learn/releases/overview#properties) for more information.
</Info>

Hit **Next** and select the config schemas to include in the release. 

<Frame>
  ![](/images/releases/create-dialog-step2.png)
</Frame>

Only one config schema per config type can be included in a release. By default, the latest schema for each config type is chosen. To select a different version, click the dropdown for the config type and select the desired version. 

To view the schema's content and metadata, hover over the schema and click the **View** button.

<Frame>
  ![](/images/releases/create-dialog-step1:view-schema-button.png)
</Frame>

Once you're satisfied with the selected schemas, hit **Create**.

## View a release

**Dashboard**

To view a release and its schemas, click into the release from the releases page and switch to the **Overview** tab at the top.

## Delete a release

Deleting a release is only possible if there exist no deployments which have been deployed to a device with that release. Otherwise, the release cannot be deleted.

To delete a release, click the ellipses (...) on the release you want to delete and hit **Delete**.

<Frame>
  ![](/images/releases/ellipses-dropdown.png)
</Frame>

A confirmation dialog will appear. Hit **Delete** to confirm the deletion.

<Frame>
  ![](/images/releases/delete-dialog.png)
</Frame>
