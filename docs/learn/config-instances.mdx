## Properties

<ParamField path="target status" type="string">

<EditableBadge />

The [desired state](/docs/learn/deployments/overview#target-status) of the config instance, inherited from the config instance's <Tooltip tip={CONFIG_INSTANCE_OWNER_TOOLTIP.tip} cta={CONFIG_INSTANCE_OWNER_TOOLTIP.cta} href={CONFIG_INSTANCE_OWNER_TOOLTIP.href}>owner</Tooltip>.

Enum: `created`, `deployed`, `removed`

</ParamField>

<ParamField path="activity status" type="string">

<MutableBadge />

The last known [activity state](/docs/learn/deployments/overview#activity-status) of the config instance, inherited from the config instance's <Tooltip tip={CONFIG_INSTANCE_OWNER_TOOLTIP.tip} cta={CONFIG_INSTANCE_OWNER_TOOLTIP.cta} href={CONFIG_INSTANCE_OWNER_TOOLTIP.href}>owner</Tooltip>.

Enum: `created`, `queued`, `deployed`, `removed`

</ParamField>

<ParamField path="error status" type="string">

<MutableBadge />

The last known [error state](/docs/learn/deployments/overview#error-status) of the config instance, inherited from the config instance's <Tooltip tip={CONFIG_INSTANCE_OWNER_TOOLTIP.tip} cta={CONFIG_INSTANCE_OWNER_TOOLTIP.cta} href={CONFIG_INSTANCE_OWNER_TOOLTIP.href}>owner</Tooltip>.

Enum: `none`, `failed`, `retrying`

</ParamField>

<ParamField path="status" type="string">

<MutableBadge />

This property merges the `activity status` and `error status`
fields, with error states taking precedence over activity states when errors
are present. 

Below are some examples of the status given different activity and error states.

| Activity Status | Error Status | Status |
|-----------------|--------------|--------|
| `created`       | `none`       | `created` |
| `deployed`      | `none`       | `deployed` |
| `queued`        | `retrying`   | `retrying` |
| `removed`       | `failed`     | `failed` |

Enum: `created`, `queued`, `deployed`, `removed`, `failed`, `retrying`

</ParamField>

<ParamField path="content" type="object">

<ImmutableBadge />

The configuration values that are deployed to the device in the format of a JSON object.

</ParamField>

<ParamField path="file path" type="string">

<ImmutableBadge />

The file path is to the config instance is deployed to relative to the `/srv/miru/config_instances` directory.

A file path of `/v1/motion-control.json` deploys the config instance to `/srv/miru/config_instances/v1/motion-control.json`.

</ParamField>

<ParamField path="config schema" type="Config Schema">

<ImmutableBadge />

The [config schema](/docs/learn/schemas/overview#properties) which the config instance must adhere to.

Examples: `SCH-FmoDN`, `SCH-3JcLq`, `SCH-HHDRn`

</ParamField>

<ParamField path="device" type="Device">

<ImmutableBadge />

The [device](/docs/learn/devices/overview#properties) which the config instance is deployed to.

Examples: `Sojourner`, `Johnny Test`, `Robot 11`

</ParamField>

<ParamField path="owner" type="Deployment">

<MutableBadge />

The deployment that a config instance currently belongs to. Visit the [owner](/docs/learn/config-instances#owner) section for more information.

Examples: `DPL-123`, `DPL-456`, `DPL-789`

</ParamField>

<ParamField path="origin" type="Deployment">

<ImmutableBadge />

The deployment with which the config instance was created. Visit the [origin](/docs/learn/config-instances#origin) section for more information.

Examples: `DPL-123`, `DPL-456`, `DPL-789`

</ParamField>

## Formats

Currently, config instances only support JSON. 

However, support for more formats including YAML and JSON-C are coming soon!
