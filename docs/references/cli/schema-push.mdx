Push a config schema to Miru. 

### Requirements

- The schema file must be committed to a local git repository
- The git repository must have a remote URL (GitHub, GitLab, Bitbucket, etc.) configured
- The schema must be annotated with its config type (see [annotations](#schema-annotations) below)

### Usage

```bash
miru schema push PATH --version VERSION
```

**Arguments**

<ParamField path="PATH" type="string" required>
The relative or absolute file system path of the schema file to push.
</ParamField>

**Flags**

<ParamField path="--version, -v" type="string" required>
The version to give the schema. Must be a valid semantic version.

Examples: `v1`, `2.0.0`, `1`
</ParamField>

<ParamField path="--allow-duplicate" type="boolean">
Allows the schema to be created even if another schema with equivalent content, but a different version, already exists for the same config type.
</ParamField>

<ParamField path="--language, -l" type="string">
The language the schema is written in. The language is auto-detected but may be overridden with this flag if needed.

Examples: `jsonschema`, `cue`
</ParamField>
