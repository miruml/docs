## Properties

<ParamField path="version">
A free-form version identifier unique per config type. Supports any versioning scheme such as SemVer, CalVer, or simple integers. Immutable once created.

Examples: `v1`, `2.0.0`, `1`, `2024-11-05`
</ParamField>

<ParamField path="language">
The schema language used to define the schema. Currently supports `jsonschema` (Draft 2020-12) and `cue`. Auto-detected from file extension.

Examples: `jsonschema`, `cue`
</ParamField>

<ParamField path="format">
The file format of the schema. Auto-detected based on file extension.

Examples: `json`, `yaml`, `cue`
</ParamField>

<ParamField path="content">
The raw schema definition containing structure, types, and constraints. Immutable once created.
</ParamField>

<ParamField path="instance_filepath">
The relative filepath where config instances will be deployed on devices. Defaults to `/{config-type-slug}.json` if not specified. This is the only mutable field.

Examples: `/v1/mobility.json`, `/config/safety.json`
</ParamField>

<ParamField path="digest">
A hash of the canonicalized schema content, automatically generated for duplicate detection. Two schemas with identical content will have the same digest.
</ParamField>
