## Creating an Override

Navigate to the `Mobility` config type and click `+ Create` on the latest config schema to open the overrides page.

<Frame>
    <img
        src="/images/getting-started/create-overrides-button.png" 
        alt="Create Overrides Button" 
    />
</Frame>

The default configuration for the `Mobility` config schema is displayed on the left. To edit the overrides, click the `Edit` button in the top right.

<Frame>
    <img
        src="/images/getting-started/overrides-editor-default.png" 
        alt="Overrides Editor Default" 
    />
</Frame>

Click the `+ Add Override` to create two overrides: `Mobility Settings` and `Telemetry Settings`.

<Tabs>
    <Tab title="Mobility Settings">
        <Frame>
            <img
        src="/images/getting-started/override-create-mobility-settings.png" 
        alt="Override Create Mobility Settings" 
            />
        </Frame>
    </Tab>
    <Tab title="Telemetry Settings">
        <Frame>
    <img
        src="/images/getting-started/override-create-telemetry-settings.png"

            />
        </Frame>
    </Tab>
</Tabs>

<Info>

## Populate the Overrides

### Mobility Settings

Click the `+ Override` button next to `max_angular_speed_radps` under the `Mobility Settings`
<Frame>
    <img
        src="/images/getting-started/override-mobility-override-button.png" 
        alt="Override Mobility Settings" 
    />
</Frame>

In the appearing modal, navigate to the `robot_model` tag type and select the `max_angular_speed_radps` field. 

This sets the `Mobility Settings` override to render its `max_angular_speed_radps` field with the value found at the `max_angular_speed_radps` field for a given tag of the `robot_model` tag type.

<Frame>
    <img
        src="/images/getting-started/override-mobility-ttf-select.png" 
        alt="Override Mobility Settings" 
    />
</Frame>

Repeat this process for the `max_linear_speed_mps` field, linking it to the `max_linear_speed_mps` field in the `robot_model` tag type.

Finally, use the corresponding metadata found in the `deployment_env` tag type to set the following fields in the `Mobility Settings` override:
- `navigation_mode` 
- `obstacle_avoidance_enabled` 
- `telemetry.upload_interval_sec`

Once all the override fields have been set, the `Mobility Settings` override should match the example below.

<Frame>
    <img
        src="/images/getting-started/override-mobility-complete.png" 
        alt="Override Mobility Settings" 
    />
</Frame>

### Telemetry Settings

For the `Telemetry Settings` override, link the `heartbeat_interval_sec` field to the `heartbeat_interval_sec` field in the `region` tag type.
        
<Frame>
    <img
        src="/images/getting-started/override-telemetry-kvp-button.png" 
        alt="Override Telemetry Settings" 
    />
</Frame>

In the appearing modal, navigate to the `region` tag type and select `heartbeat_interval_sec`.

<Frame>
    <img
        src="/images/getting-started/override-telemetry-ttf-select.png" 
        alt="Override Telemetry Settings" 
    />
</Frame>

Repeat this process by setting the `telemetry.upload_interval_sec` field in the `Telemetry Settings` override to the `telemetry_upload_interval_sec` field found in the `region` tag type.

Once all the override fields have been set, the `Telemetry Settings` override should match the example below.
<Frame>
    <img
        src="/images/getting-started/override-telemetry-complete.png" 
        alt="Override Telemetry Settings" 
    />
</Frame>


## Render the Configuration

After defining the `Mobility Settings` and `Telemetry Settings` overrides, click 
`Done` to switch back to render mode.

<Frame>
    <img
        src="/images/getting-started/overrides-list-render-mode.png" 
        alt="Overrides List" 
    />
</Frame>

Select the `Render` button at the top left of the page to open the dropdown menu.

<Frame>
    <img
        src="/images/getting-started/overrides-render-menu.png" 
        alt="Overrides Render Menu" 
    />
</Frame>

Select `Robot 1` as the device. The tags of `Robot 1` are used by the overrides to render the final configuration.

<Frame>
    <img
        src="/images/getting-started/overrides-render-menu-device-selected.png" 
        alt="Overrides Render Menu Device Selected" 
    />
</Frame>

Click `Render` to view the config instance that would be deployed to `Robot 1` given this particular config schema.

<Frame>
    <img
        src="/images/getting-started/override-config-instance.png" 
        alt="Override Config Instance" 
    />
</Frame>
