## Creating an Override

Navigate to the `Mobility` config and click `+ Create` to open the overrides page.

<Frame>
    <img
        src="/images/getting-started/create-overrides-button.png" 
        alt="Create Overrides Button" 
    />
</Frame>

You'll see the default configuration for the `Mobility` config schema. To create 
an override, click the `Edit` button (to the right) and 
you should see the following screen.

<Frame>
    <img
        src="/images/getting-started/overrides-editor-default.png" 
        alt="Overrides Editor Default" 
    />
</Frame>

Click the `+ Add Override` to create these two overrides.

<Tabs>
    <Tab title="Mobility Settings">
        <Frame>
            <img
        src="/images/getting-started/override-create-mobility-settings.png" 
        alt="Override Create Mobility Settings" 
            />
        </Frame>
    </Tab>
    <Tab title="Telemetry Settings">
        <Frame>
    <img
        src="/images/getting-started/override-create-telemetry-settings.png" 
        alt="Override Create Telemetry Settings" 
            />
        </Frame>
    </Tab>
</Tabs>

<Info>

Here is the final screen after creating the overrides.

## Defining the Overrides

### Mobility Settings

1. Click the `+ Override` button next to `max_angular_speed_radps`.
        <Frame>
            <img
                src="/images/getting-started/override-mobility-override-button.png" 
                alt="Override Mobility Settings" 
            />
        </Frame>

        2. In the modal that appears, navigate to the `robot_model` tag type and select `max_angular_speed_radps` to override.

        <Frame>
            <img
                src="/images/getting-started/override-mobility-ttf-select.png" 
                alt="Override Mobility Settings" 
            />
        </Frame>

        3. Repeat this process for `max_linear_speed_mps`, also found under the `robot_model` tag type.

        4. For the remaining settings, you'll use the corresponding metadata found in the 
        `deployment_env` tag type to override the following key-value pairs:
           - `navigation_mode` 
           - `obstacle_avoidance_enabled` 
           - `telemetry.upload_interval_sec`


        Once all the key-value pairs have been set, the `Mobility Settings` override 
        should match the example below.

        <Frame>
            <img
                src="/images/getting-started/override-mobility-complete.png" 
                alt="Override Mobility Settings" 
            />
        </Frame>

### Telemetry Settings
1. Click the `+ Override` button next to `heartbeat_interval_sec`.
        
        <Frame>
            <img
                src="/images/getting-started/override-telemetry-kvp-button.png" 
                alt="Override Telemetry Settings" 
            />
        </Frame>
        
        2. In the modal that appears, navigate to the `region` tag type and select `heartbeat_interval_sec`.

        <Frame>
            <img
                src="/images/getting-started/override-telemetry-ttf-select.png" 
                alt="Override Telemetry Settings" 
            />
        </Frame>
        
        3. Repeat this process by overriding `telemetry.upload_interval_sec` in the Config Schema
        with the `telemetry_upload_interval_sec` metadata found in the `region` tag type.

        Once all the key-value pairs have been set, the `Telemetry Settings` override should match the example below.
        <Frame>
            <img
                src="/images/getting-started/override-telemetry-complete.png" 
                alt="Override Telemetry Settings" 
            />
        </Frame>


## Rendering Final Config

After defining the `Mobility Settings` and `Telemetry Settings` overrides, click 
`Done` to switch back to render mode.

<Frame>
    <img
        src="/images/getting-started/overrides-list-render-mode.png" 
        alt="Overrides List" 
    />
</Frame>

Click the `Render` button at the top of the page to open the dropdown menu.

<Frame>
    <img
        src="/images/getting-started/overrides-render-menu.png" 
        alt="Overrides Render Menu" 
    />
</Frame>

Select `Robot 1` as the client. The tags of the client will be used in the overrides to 
determine the final configuration.

<Frame>
    <img
        src="/images/getting-started/overrides-render-menu-client-selected.png" 
        alt="Overrides Render Menu Client Selected" 
    />
</Frame>

Click `Render` and to view the Concrete Config that will be deployed to the client.

<Frame>
    <img
        src="/images/getting-started/override-concrete-config.png" 
        alt="Override Concrete Config" 
    />
</Frame>

Congratulations! You've successfully defined overrides for the `Mobility` Config.

<Frame>
    <img
        src="/images/getting-started/overrides-rendered-concrete-config.png" 
        alt="Override Concrete Config" 
    />
</Frame>
