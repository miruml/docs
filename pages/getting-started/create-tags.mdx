## Tag Types
Miru uses [tag types](/pages/vocabulary#tag-types-%26-tags) to organize related tags that share a metadata schema.


### Create Tag Types
For this guide, you’ll create three tag types
- Customer
- Environment
- Hardware

Click `+ New Tag Type` to open the create tag type form to get started.

### Metadata Schema
Each tag type can have a metadata schema that defines required fields for all tags 
of that type. This ensures consistent metadata for tags when used in overrides.

Here are the metadata schemas for the three tag types we’re creating:

<CodeGroup>

```yaml customer
title: Customer Metadata
type: object
properties:
  telemetry_interval_sec:
    type: integer
required:
  - telemetry_interval_sec
```

```yaml environment
title: Environment Metadata
type: object
properties:
  telemetry_interval_sec:
    type: integer
  emergency_stop_sensitivity:
    type: number
    minimum: 0.0
    maximum: 1.0
required:
  - telemetry_interval_sec
```

```yaml hardware
title: Hardware Metadata
type: object
properties:
  payload_capacity:
    type: number
  default_speed_mode:
    type: string
    enum: [slow, normal, fast]
required:
  - payload_capacity
  - default_speed_mode
```

</CodeGroup>

You can **upload** these schemas from the [tag-types directory](https://github.com/miruml/getting-started/tree/main/tag-types) 
in the getting-started repository, or define and edit them directly in the UI.

## Tags
Now we can create tags to categorize clients and define metadata for overrides.

### Create Tags
Click `+ New Tag` to open the create tag form.

<Frame>
    <img
        src="/images/getting-started/create-tag-page.png" 
        alt="Create Tag Page" 
    />
</Frame>

### Setting Metadata

Use the following values when creating your tags.

<CodeGroup>

```yaml customer
# Tag: Standard
telemetry_interval_sec: 60

# Tag: Premium
telemetry_interval_sec: 30
```

```yaml environment
# Tag: Development
emergency_stop_sensitivity: 0.9
telemetry_interval_sec: 45

# Tag: Production
emergency_stop_sensitivity: 0.6
telemetry_interval_sec: 120
```

```yaml hardware
# Tag: Raspberry Pi 4 Model B
default_speed_mode: slow
payload_capacity: 5.0

# Tag: NVIDIA Jetson Orin Nano
default_speed_mode: normal
payload_capacity: 10.0

# Tag: NVIDIA Jetson Orin AGX
default_speed_mode: fast
payload_capacity: 20.0
```

</CodeGroup>
