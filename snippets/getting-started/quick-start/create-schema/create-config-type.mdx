## Create a config type

Before defining a schema, we first need to create a config type to house it. Navigate to the [config types page](https://app.miruml.com/configs).

<Frame>
  ![](/images/config-types/create-page.png)
</Frame>

Click **New Config Type** button. Supply the name `Mobility` and slug `mobility`. Finally, click **Create**.

<Frame>
   ![](/images/config-types/create-dialog.png)
</Frame>

<Warning>
    Config type slugs are permanent and cannot be changed after creation.
</Warning>

The slug is a unique identifier for the config type--it's how the CLI determines what config type a schema belongs to. As such, config schemas *must* be annotated with a config type slug.
